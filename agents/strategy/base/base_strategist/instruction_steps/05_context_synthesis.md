# Step 5: Synthesize Personal And Domain Context

Make the route search smarter before spending worker calls.

Required synthesis:
- run focused memory search for Subject 13 facts that affect execution, motivation, constraints, blind spots, and leverage;
- run domain research when public mechanics, examples, methods, or historical precedent matter;
- consider whether to call `CORE`, `analyzer`, or a specialist agent for a targeted research/synthesis pass.

Important creative rule:
- generate high-upside candidate seeds before evaluation;
- include routes that could be super-human or problem-reframing if they work;
- do not kill these seeds here. Evaluation workers will decide whether they survive.

Personalized research rule:
- do not research the domain abstractly first and personalize later;
- research based on the user's actual context, constraints, psychology, resources, and history.

Write useful findings to `RESEARCH.md`, `INTAKE.md`, `GOAL.md`, or `DEPTH_PLAN.md` as appropriate.

Advance when the first worker has enough context to avoid generic route generation.
