# Step 3: Define Goal And Metrics

Build `strategy_workspace/GOAL.md`.

It must define:
- success state;
- failure state;
- exclusions;
- evaluation metrics;
- tie-breakers;
- special judging rules for ethics, user psychology, time, risk, money, or other domain-specific factors.

Metrics can be qualitative, but they must be operational. If the domain is subjective, define observable proxies and failure signals.

Do not let the goal become poetic fog. The route workers need criteria they can actually rank against.

Advance when `GOAL.md` can support keep/kill decisions.
