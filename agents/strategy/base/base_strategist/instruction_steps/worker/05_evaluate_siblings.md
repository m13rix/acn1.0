# Worker Step 5: Evaluate Siblings With Evidence

This is the core quality step. Do not make it shallow.

For each route, write an eval file with:
- `Decision`
- `Rank among siblings`
- `Question this depth is answering`
- `Evidence sources used`
- `Quantitative model`
- `What-if checks`
- `Main unknowns`
- `Confidence`
- `Why keep or kill is justified`
- `Next-question check`

Evaluation stack:
1. facts;
2. derived estimates;
3. uncertainty/world-branch model where useful;
4. decision.

For quantitative domains, use comparable units:
- expected payoff per minute/hour/week;
- expected success probability;
- expected time to target;
- downside distribution;
- sensitivity ranges.

For qualitative/personal domains, use operational proxies:
- likelihood of execution by Subject 13;
- emotional cost;
- reversibility;
- social risk;
- information value;
- chance of unlocking a much better route;
- failure signs.
- etc.

Before killing a route, ask:
- What if one permission flips it?
- What if stronger preparation makes it viable?
- What if it is bad as a final route but excellent as an information-gathering opener?
- What if this route works and completely reframes the problem?

Use `strategy.paths/events` for uncertain environment outcomes when it clarifies ranking. Do not use it as ceremony.

CRUCIAL: ALWAYS KEEP ONLY ONE ROUTE ALIVE, kill others, it is the strongest and the one with the most potential and practical possiblity that stays

Advance when every sibling has a real evidence-backed evaluation.
