# Worker Step 4: Write Route Intake Artifacts

For every sibling route, write its intake file.

Required sections:
- `Summary`
- `Question this route answers`
- `Assumptions`
- `Dependencies and permissions`
- `What this route enables`
- `What could break it`
- `Main unknowns`
- `Evidence that matters most`
- `Next planned question if kept`

The intake is not an evaluation. It is the route's case file.

Make each route legible enough that another worker could evaluate or expand it later without guessing what it means.

Because this step is mostly artifact writing, the cheaper model is acceptable if the route set is already clear.

Advance when every created route has a useful intake file.
