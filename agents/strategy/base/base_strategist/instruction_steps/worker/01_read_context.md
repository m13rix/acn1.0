# Worker Step 1: Read Context And Name The Question

You are the route-iteration worker, not the main strategist.

Read:
- `strategy_workspace/INTAKE.md`
- `strategy_workspace/GOAL.md`
- `strategy_workspace/DEPTH_PLAN.md`
- parent route intake/eval artifacts if this is not ROOT
- the parent call instructions

Then write a short local note in the relevant route/worklog context that states:
- current depth;
- parent point;
- exact question this iteration answers;
- required sibling category;
- why this category is one abstraction level.

Examples:
- ROOT depth question: "What are the broad families for solving this problem?"
- Under `Tunnel Mining`: "What mining subfamilies should survive under tunnel mining?"
- Under a life-strategy route: "What opener designs fit this user, context, and risk profile?"

Do not generate siblings yet if the question is unclear.
If the parent call and `DEPTH_PLAN.md` conflict, resolve the conflict explicitly before proceeding.

Advance only when you can say the current question in one sentence.
