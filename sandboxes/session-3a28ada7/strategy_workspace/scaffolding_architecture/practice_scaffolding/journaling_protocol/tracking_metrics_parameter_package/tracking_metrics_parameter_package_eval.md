### Key metrics and targets
- **Journaling Consistency:** Baseline = unknown; Target = >5 days/week within 4 weeks.
- **Emotional Granularity Score:** Baseline = broad labels; Target = >5 distinct emotion words per entry by Week 4 (requires content analysis opt-in).
- **Self-Compassion Ratio:** Baseline = unknown; Target = >1:2 self-compassionate vs self-critical statements by Week 4 (requires content analysis opt-in).
- **Entry Length:** Baseline = unknown; Target = 20-200 words median (screens for under/over-engagement).
- **Red-flag threshold:** 0 entries for 5+ consecutive days during non-break periods triggers protocol pause/check-in.

### Sensitivity analysis
- If Maxim opts in to content analysis, score increases by ~0.8 (emotional granularity and self-compassion become measurable).
- If Maxim finds weekly dashboards anxiety-producing, score drops by ~1.0 (tracking becomes counterproductive).
- If tracking feels like surveillance, score drops by ~1.5 (protocol-killer risk).
- If metric gaming occurs (minimal entries to "hit the metric"), score drops by ~0.7.

## What-if checks

**What if Maxim refuses content analysis?**
→ Fallback to metadata-only tracking (consistency, entry length, timestamp). Still valuable but loses emotional granularity and self-compassion measurement.

**What if weekly dashboards create performance anxiety?**
→ Reduce frequency to biweekly or monthly. Or switch from "dashboard" to "milestone celebration" framing (positive-only, no "needs attention" status).

**What if tracking becomes an intellectualization loop?**
→ Cap the number of tracked metrics at 3. Remove any metric that Maxim starts optimizing for rather than using as gentle feedback.

**What if this works and metrics prove the protocol is effective?**
→ The metrics become the evidence that justifies expanding the Journaling Protocol to other areas, or justifies its obsolescence ("You hit all targets, you don't need this anymore").

## Main unknowns
1. Whether Maxim will opt in to content analysis — this is the single biggest variable.
2. Whether metric feedback is motivating or anxiety-producing for Maxim.
3. The correct red-flag thresholds for a 15-year-old with motivation volatility.
4. Whether Maxim prefers Telegram-based dashboards, text summaries, or no dashboard at all.

## Confidence
**Moderate (65%)** — The package has strong metric alignment but significant execution risk. Content analysis opt-in is uncertain. Surveillance perception is a real threat. The package is valuable but not essential for protocol launch.

## Why kill is justified (Rank 4)
1. **Execution risk is high:** The most valuable metrics (emotional granularity, self-compassion ratio) require content analysis opt-in, which is uncertain. Without opt-in, the package reduces to "count entries and words" — useful but not worth a standalone depth-6 expansion.
2. **Surveillance threat:** Even metadata tracking may feel like surveillance to a privacy-sensitive 15-year-old. The risk of triggering disengagement outweighs the information value.
3. **Embedded in other packages:** Consistency tracking is a simple binary that can be built into the Format & Delivery package (auto-timestamp on entry). Red-flag thresholds can be defined in the Prompt Package as "skip protocol" rules. The Tracking package does not need standalone depth-6 expansion.
4. **Information value is delayed:** We only learn whether metrics are motivating or anxiety-producing after weeks of data. The Prompt Package reveals preferences in days.
5. **Lowest total score:** 6.27 vs 8.70 (Prompt), 7.25 (Format), 6.75 (Obsolescence). The gap to Prompt is significant.

## What-if checks before killing
**What if one permission flips it?**
→ If Maxim enthusiastically opts in to content analysis, the package's value increases significantly. But this is unknown and cannot be assumed. The package can be reactivated if opt-in occurs.

**What if stronger preparation makes it viable?**
→ The package IS viable — it is well-designed. The question is priority. It can be folded into the surviving Prompt Package as its "feedback loop" component.

**What if it is bad as a final route but excellent as an information-gathering opener?**
→ We learn whether tracking is motivating by observing Maxim's response to the basic consistency metric built into the Format Package. No additional information-gathering value from standalone expansion.

**What if this route works and completely reframes the problem?**
→ Unlikely. Tracking is a measurement layer, not a reframing layer. It supports rather than redefines the protocol.
