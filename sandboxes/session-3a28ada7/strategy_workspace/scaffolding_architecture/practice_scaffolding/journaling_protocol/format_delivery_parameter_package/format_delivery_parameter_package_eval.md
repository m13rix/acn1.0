## What-if checks

**What if Maxim prefers physical journaling?**
→ Format package accommodates this: physical notebook with same prompt printed on a card or written by Maxim himself. Digital remains backup.

**What if the scheduled time conflicts with Maxim's routine?**
→ Time is adjustable within the first 2 weeks. The protocol includes a "time calibration" phase where 3 different times are tested and the one with highest completion rate wins.

**What if Maxim feels tracked even by metadata?**
→ Explicit transparency: show Maxim exactly what is tracked (timestamp, length) and what is NOT (content). Offer opt-out of all tracking except binary completion.

**What if format flexibility becomes a source of decision fatigue?**
→ Lock format for 2-week periods. "Choose once, stick for 2 weeks, then re-evaluate." Reduces daily decision load.

## Main unknowns
1. Maxim's preferred format — digital vs. physical.
2. Maxim's optimal journaling time — evening pre-20:00, but exactly when?
3. Smartwatch capabilities — can it display a full prompt or just a notification?
4. Maxim's privacy comfort level — does he want complete wall or option to share?

## Confidence
**Moderate-High (75%)** — Format is a necessary enabler but not the primary value driver. The package is well-designed and addresses known risks (privacy, flexibility, timing), but its success depends heavily on Maxim's personal preferences which are currently unknown.

## Why keep is justified (Rank 2, temporary survivor)
1. **Necessary enabler:** Prompts need a delivery mechanism. Without format specification, the protocol cannot execute.
2. **Friction reduction:** Correct format and timing can make the difference between a habit that sticks and one that dies. The evidence on after-20:00 energy collapse makes timing critical.
3. **Privacy protection:** This is the package that defines privacy safeguards — a GOAL.md requirement. Strong privacy design prevents the surveillance failure state.
4. **Format flexibility:** The digital/physical hybrid approach matches Maxim's demonstrated preference for both efficiency and authenticity.
5. **Information value:** Testing format preferences reveals critical data about Maxim's habits that informs not just journaling but all future Project Arete interventions.

## Why this is a TEMPORARY survivor (not co-equal with Rank 1)
- The Prompt Parameter Package is the content; Format & Delivery is the container. If resources are constrained, the prompt content can be delivered through ANY channel (even a sticky note). Format is important but secondary.
- Format decisions can be made empirically in the first 2 weeks of protocol launch — they do not need a full depth-6 expansion before the protocol starts.
- If the Prompt Package fails, Format & Delivery cannot save it. The reverse is not true.
