### Sensitivity analysis
- If Maxim's habit formation speed is at the fast end (18 days), score increases by ~0.5 (obsolescence can be accelerated).
- If Maxim resists planned obsolescence framing, score drops by ~1.0 (protocol may feel like abandonment).
- If relapse re-engagement is too aggressive, score drops by ~0.5 (may create dependency loop).
- If the protocol never reaches consistency thresholds, this package becomes irrelevant — obsolescence requires a functioning habit first.

## What-if checks

**What if Maxim embraces obsolescence enthusiastically?**
→ Best case. Accelerate timeline. Celebrate independence as system success.

**What if Maxim resists transitions as "the system leaving me"?**
→ Reframe as "leveling up" rather than "being abandoned." Each transition is a graduation, not a loss.

**What if the habit never forms well enough to justify obsolescence?**
→ This package is deferred. The protocol stays in Phase 1 until consistency thresholds are met. Obsolescence is conditional on habit formation.

**What if this works perfectly and journaling becomes fully independent?**
→ This is the exact success state defined in GOAL.md. The Obsolescence Package has achieved its purpose.

## Main unknowns
1. Maxim's actual habit formation speed — fast (18 days) or slow (254 days)?
2. Whether Maxim will embrace or resist planned obsolescence framing.
3. Whether relapse re-engagement should be automatic or manual.
4. What "graduation" ritual would feel meaningful to Maxim.

## Confidence
**Moderate (70%)** — The package is well-designed and aligns strongly with the Scaffolding model's core principle. However, it is a SECONDARY concern: obsolescence only matters if the habit forms in the first place. Its slow feedback loop (8-12 weeks) and conditional nature make it less urgent than Prompt or Format packages.

## Why kill is justified (Rank 3)
1. **Conditional on prior success:** Obsolescence is irrelevant if the Journaling Protocol never achieves consistency. The Prompt and Format packages must work FIRST.
2. **Slow feedback loop:** Results only visible after 8-12 weeks. The Prompt Package gives feedback in 1-2 weeks.
3. **Embedded in other packages:** The Obsolescence timeline can be defined as a subsection within the Prompt Package (prompt evolution schedule) and the Format Package (template evolution). It does not need to be a standalone parameter package at this depth.
4. **Lower information value:** What we learn from Obsolescence (habit formation speed) is valuable but only after weeks of data. Prompt and Format packages reveal preferences within days.
5. **Resource efficiency:** Keeping 3+ routes alive at depth 5 is expensive. The Obsolescence parameters can be folded into the surviving Prompt Package as its "evolution timeline" component.

## What-if checks before killing
**What if stronger preparation makes it viable?**
→ The package IS viable — it is well-designed. The question is whether it needs standalone depth-6 expansion NOW. It does not. Its parameters can be defined within the Prompt Package's evolution schedule.

**What if it is bad as a final route but excellent as an information-gathering opener?**
→ We already know the key information needed (habit formation timeline from Lally/Singh). Maxim-specific speed will be revealed by the Prompt Package's consistency tracking. No additional information-gathering value from standalone expansion.

**What if this route works and completely reframes the problem?**
→ Unlikely. Obsolescence is a well-understood concept in behavioral design. It supports rather than reframes the Journaling Protocol.
