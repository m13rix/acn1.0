# INTAKE — Obsolescence & Transition Parameter Package

## Summary
The planned obsolescence path for the Journaling Protocol: checkpoint schedule for reducing AI dependency, transition triggers (consistency thresholds), independence criteria, and post-obsolescence continuation mode. This is the "exit strategy" — the parameter package that defines how the scaffold is dismantled.

## Question this route answers
"How does the Journaling Protocol transition from AI-prompted → self-prompted → independent practice, and what are the measurable triggers for each transition?"

Position in family: Depth 5, parameter package 3 of 4 under Journaling Protocol. Siblings: Prompt Parameter Package, Format & Delivery, Tracking & Metrics.

## Assumptions
- The Scaffolding model (Depth 2 winner) requires explicit obsolescence for every sub-component. Journaling Protocol is the first practice scaffold to be dismantled.
- Habit formation research (Lally 2009, Singh 2024 meta-analysis) shows 66 days median for automaticity, range 18-254 days. The protocol should plan for 8 weeks (56 days) as the primary obsolescence window, with 12 weeks (84 days) as the extended window.
- Maxim learns fast (systems thinker, rapid skill acquisition) — prompt fading can be more aggressive than typical adolescent timeline.
- Consistency (5+ entries/week) is the primary trigger for transition, not emotional depth or entry quality.
- After obsolescence, Maxim may continue journaling independently — this is the best case outcome (GOAL.md: system obsolescence).
- A "relapse" mechanism is needed: if Maxim stops journaling after obsolescence, the system can re-engage at the appropriate earlier stage.

## Dependencies and permissions
- Permission to reduce prompt frequency and change delivery format over time (explicitly communicated to Maxim upfront).
- Permission to track transition-relevant metrics (consistency, self-prompting rate).
- Permission to "check in" after obsolescence (weekly then monthly then quarterly) to verify independence persists.
- Maxim's understanding and agreement that this is planned obsolescence — the goal is to not need the system.

## What this route enables
- **Prompt fade schedule:** Structured reduction from full prompt (Week 1-2) → partial cue (Week 3-4) → minimal contextual cue (Week 5-6) → no prompt (Week 7-8). Based on prompt-fading best practices.
- **Transition triggers:** Measurable thresholds for moving between phases. Primary: 5+ entries/week for 2 consecutive weeks. Fallback: time-based if consistency is borderline.
- **Independence criteria:** Clear definition of "independent journaling" — 5+ entries/week for 4 consecutive weeks without any AI prompt, with at least 2 entries showing self-generated topics.
- **Relapse detection and re-engagement:** If consistency drops below 3 entries/week for 2 consecutive weeks post-obsolescence, system offers optional re-engagement at the phase appropriate to current consistency.
- **Post-obsolescence continuation mode:** If Maxim continues journaling independently, the only remaining system touchpoint is a monthly check-in ("Still journaling?") that fades to quarterly to zero.
- **Obsolescence celebration:** When independence criteria are met, a deliberate "graduation" moment that frames this as success (Maxim outgrew the scaffold).

## What could break it
- **Premature independence:** Maxim may declare himself "independent" too early and stop journaling before the habit is internalized. The protocol must use objective triggers, not self-assessment.
- **Prompt fading too fast:** Research (MapHabit) shows 34% of cases where dependency increases happen because prompts were removed too quickly.
- **Maxim rejects the concept:** If Maxim sees planned obsolescence as "the system abandoning me," he may resist transitions.
- **Identity attachment:** If journaling becomes part of Maxim's identity ("I'm someone who journals"), the transition away from AI-prompts may feel like losing part of himself.
- **Relapse cycle:** Maxim may cycle through independence → relapse → re-engagement → independence repeatedly, never fully internalizing the habit.

## Main unknowns
1. Maxim's actual habit formation speed — is he in the 18-day or 254-day range of the Lally study?
2. Whether Maxim will resist the planned obsolescence framing or embrace it (systems designer by nature — likely embrace).
3. Whether relapse re-engagement should be automatic (system detects drop, re-engages) or manual (Maxim chooses to restart).
4. What "graduation" looks like — a notification? A certificate? A ritual?

## Evidence that matters most
- **Web (Lally 2009):** Mean 66 days for habit automaticity. Range 18-254 days. Missing days does not harm formation if resumed.
- **Web (Singh 2024 meta-analysis):** 2-5 months for most health behaviors. Confirms ~8 week initial window.
- **Web (prompt fading research):** Skills learned with proper prompt fading persist 89% after 6 months vs 45% without. Gradual fading is critical.
- **Web (prompt fading speed risk):** 34% of dependency increases happen when prompts removed too quickly.
- **Parent intake (obsolescence):** AI-prompted → self-prompted → independent — the three-stage path is explicitly defined.
- **GOAL.md (Obsolescence Principle):** Tie-breaker #1 — route that reduces long-term dependency wins.
- **GOAL.md (AI Dependency Score):** <2 prompts/day by month 6 — journaling prompt reduction directly contributes.
- **Memory (Maxim's learning speed):** "Rapid skill acquisition when intrinsically motivated" — supports more aggressive prompt fade.
