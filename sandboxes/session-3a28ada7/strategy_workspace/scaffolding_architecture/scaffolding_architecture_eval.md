## What-if checks

**What if Maxim resists dismantling tools he built?**
→ This is a real risk. Mitigation: frame dismantling as *completion*, not loss. "The scaffolding is removed because the building stands." Use his proof-of-change engine — each dismantled tool is proof he can function without it.

**What if the obsolescence schedule is too aggressive?**
→ Schedule is adjustable. DEPTH_PLAN.md explicitly allows acceleration/deceleration based on performance. If Maxim learns faster, remove scaffolding faster. If slower, extend.

**What if this works and completely reframes the problem?**
→ Best case: Maxim internalizes the obsolescence meta-skill and applies it to other systems in his life (school, relationships). This is the ultimate success state.

**What if one permission flips it?**
→ If transcription system becomes real-time and perfect, Mirror model gains. But Scaffolding can *incorporate* Mirror as a scaffolding layer — they are not mutually exclusive.

## Why keep is justified
1. **Direct goal alignment:** GOAL.md's #1 tie-breaker is obsolescence. Scaffolding is the only model that *embeds* obsolescence into its architecture.
2. **Psychological fit:** Maxim's cognitive cycle already includes "design → crash → meta-analyze." Scaffolding gives him a structured way to break this cycle by making each tool temporary by design.
3. **Measurable success:** The obsolescence schedule creates natural checkpoints (Week 2, Week 4, Week 8) where we can measure dependency reduction.
4. **Prevents failure states:** The dependency trap and intellectualization loop (GOAL.md failure states) are structurally prevented because tools are designed to disappear.
5. **Composability:** Scaffolding can incorporate Teacher content (Socratic questions as scaffolding layer), Ecosystem tools (each tool is a scaffolding plank), and Crisis safety (crisis protocol as first scaffolding layer to remove last).
