# GOAL

## Success state
A strategy that lets the user complete Minecraft Java Edition 1.21.11 on normal difficulty, with keepInventory off, as quickly as possible for their actual skill level, while remaining realistically executable and stable enough that the user can learn and apply it.

## Failure state
- The strategy is not executable by the user
- The strategy assumes skills the user does not have and is therefore too hard to execute
- The strategy is slower than a safer alternative that fits the user's skill
- The strategy depends on cheats/commands or other excluded methods
- The strategy ignores the user's lack of full-completion experience
- The strategy requires excessive preparation that stops being fun / practical for the user

## What counts in evaluation
Primary metric:
- Expected real-world time to full completion from world start to dragon death / game clear.

Secondary metrics:
- Execution reliability / consistency for this user
- Time cost of setup and practice before the run
- Risk of catastrophic loss from death or bad route execution
- Sensitivity to poor RNG, bad spawn, or weak mechanics execution
- Cognitive load / inventory-management burden
- Compatibility with a chosen seed and external scouting tools

## What does not count
- Style points
- Pure theoretical speed if it is too hard for the user to execute
- Routes that require commands/cheats
- Strategies optimized for multiplayer coordination unless the user explicitly wants that
- Practice time itself, except insofar as it reveals skill or improves confidence

## Tie-breakers
If two routes have similar expected time, prefer the one with:
1. higher execution reliability for this user
2. lower cognitive load
3. lower reset risk
4. better compatibility with beginner-to-intermediate skill
5. smaller practice burden

## Special judging instructions
- Because the user has not fully completed the game before, a 'faster but too hard' route should usually lose to a slightly slower route that is much more executable.
- Skill uncertainty must be handled by testing and questions, not by assumption.
- If later evidence shows the user can handle specific advanced techniques, update the ranking accordingly.
- Routes may be compared both as random-seed plans and as set-seed plans if the user chooses a seed in advance.
