# INTAKE — Seed-Map Scout-and-Set

## Exact route question
Can the user do even better than a pre-verified seed by scouting a custom world with seed maps or structure finders before the run?

## Why this route exists
- External seed-map tools are allowed.
- A tailored world could reduce travel distance and make the opener easier.
- For a beginner, a very favorable world can sometimes compensate for weaker execution.

## Assumptions
- The user can obtain a seed before starting the real run.
- Seed-map tooling is used only for planning, not cheating in-run.
- The chosen world is still meant to be playable by a human, not just theoretically optimal.

## Dependencies
- Accurate structure lookup.
- Good judgment about whether a seed is actually easier, not merely prettier on paper.
- Willingness to spend setup time before the run.

## What could break it
- Scouting takes too long.
- The selected world is only marginally better than the pre-verified alternative.
- Structure-map inaccuracies or version mismatches lead to a misleading choice.

## Information still needed
- How much the user values setup time versus in-run simplicity.
- Whether the user wants the absolute easiest opener or the quickest path to a startable run.
