# INTAKE

## Exact task
Design the best possible strategy for completing Minecraft Java Edition 1.21.11 as fast as possible on normal difficulty, without keepInventory, personalized to the user's actual skill.

## Known constraints
- Game: Minecraft Java Edition 1.21.11
- Difficulty: normal
- keepInventory: off
- Goal: full game completion, as fast as possible
- Strategy may use external non-cheat tools like seed maps
- Strategy should account for the user's actual skill level, which is initially unknown
- User has not fully completed the game before
- User wants a strategy that is fast but still realistically executable, preferably stable enough to succeed on roughly the first attempt
- The user is open to short practice/tests, and test time does not count toward the run
- The user is willing to use a chosen seed / set seed if beneficial
- The user is playing single-player in a new world

## What the user already said
- They have partially played with friends and/or with keepInventory, but not a full solo-style clear under these constraints.
- They want every detail considered: route, tools, food, and general planning.
- They are open to questions and to skill tests.
- They specifically requested that skill be measured through tests rather than guessed.

## Unknowns to resolve
- Exact skill level in combat, movement, inventory management, Nether navigation, and End fight
- Whether the user can reliably execute water clutches, basic PvE, and End fight mechanics
- Whether the user is willing to practice specific techniques before the run
- The best route family given the user's measured skill: safer overworld-first, strong structure scouting, set-seed route, bartering-heavy route, etc.
- Whether the user wants a fully seeded route or a seed-agnostic fallback plan

## Allowed tools and methods
- In-game play without commands/cheats
- External information tools such as seed maps and structure finders
- Practice runs, drills, and testing
- Research into current mechanics and route options
- Chosen seed / set seed if it improves reliability and speed

## Forbidden tools and methods
- Commands / cheats / creative mode
- Anything that would count as cheating rather than planning or scouting
- Assumptions that the user has speedrun-level skill without evidence

## Exclusions
- No divergence / monitoring work in this version
- No framework/debugging tasks
- Multiplayer optimization unless the user later asks for it

## Notes
- The eventual strategy should be personalized by skill tier, not generic.
- Early work should focus on measuring the user's current capability and then selecting a route family accordingly.
- If a skill question matters, a short test is preferred over guesswork.
