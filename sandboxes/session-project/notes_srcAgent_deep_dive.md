### A. Память и сохранение состояния

В системе реализована многоуровневая гибридная модель памяти:

1.  **Skills (Семантическая контекстная память):**
    *   **Механизм:** Использует **LanceDB** для векторного поиска. Хранит фрагменты знаний («навыки») с примерами запросов.
    *   **Принцип:** При каждом ходе сессия (`Session.ts`) выполняет семантический поиск по истории сообщений. Найденные совпадения (выше порога ~82%) **динамически внедряются в System Prompt**.
    *   **Статус:** Подтверждено кодом (`src/skills_system/SkillsService.ts`). Это механизм «подгрузки» релевантных инструкций «на лету».

2.  **MemoryService (Графовая долговременная память / LTM):**
    *   **Механизм:** Сложная система «фактов» (facts) и «связей» (links). Поддерживает автоматическое связывание (auto-linking) новых фактов с существующими на основе семантической близости и LLM-анализа.
    *   **Принцип:** Умеет индексировать документы (.md, .txt), извлекать из них атомарные факты и выстраивать граф знаний. Поиск выполняется по цепочкам графа.
    *   **Статус:** Подтверждено кодом (`src/memory_system/MemoryService.ts`). Это полноценная **LTM**.

3.  **Notepad (Явная декларативная память):**
    *   **Механизм:** Простое JSON-хранилище (`data/notepad/notes.json`).
    *   **Принцип:** Агент может явно записывать, читать и редактировать заметки. Это аналог «рабочей тетради» для хранения структурированной информации, которую агент считает важной сохранить в явном виде.

4.  **Session State (Кратковременная память / STM):**
    *   **Механизм:** Хранится в оперативной памяти в объекте `Session`. Включает историю текущего диалога и временный контекст файлов.
    *   **Разграничение:** В системе четко видно разделение на:
        *   **STM:** История текущей сессии (`Session.ts`).
        *   **LTM (Implicit):** Граф фактов в `MemoryService`.
        *   **LTM (Explicit):** Заметки в `Notepad` и семантические навыки в `Skills`.

### B. Heartbeat / Event-driven архитектура

Это **не просто cron**, а полноценная система проактивности.

*   **Bindings и Sensors:** Сенсоры — это модули, генерирующие события (например, `clock`, `file-watcher`). Биндинги (`bindings.json`) — это «подписки», связывающие событие с исполняемым JS-кодом (обработчиком).
*   **Реализация:** Когда сенсор триггерит событие, `HeartbeatService` находит подходящие биндинги и запускает их в **изолированной песочнице** (`LocalSandbox`). Обработчики имеют доступ к инструментам (tools), что позволяет агенту совершать действия (отправлять сообщения, запускать поиск) без прямого запроса пользователя.
*   **Ограничения:** Проактивность ограничена наличием запущенных сенсоров. На данный момент основная активность завязана на `clock.schedule` (гибкое расписание) и внешние триггеры.

### C. Strategy System

Наиболее сложный аналитический компонент.

*   **Модель данных:** Граф состояний. Объекты `StrategyProject` -> `StrategyPath` -> `Nodes` (состояния) + `Events` (переходы).
*   **Аналитика:** Система умеет математически (марковский анализ) вычислять:
    *   Вероятность достижения цели (`goal`) или провала (`fail`).
    *   Ожидаемую полезность (`stateDelta`) пути.
    *   Математическое ожидание количества шагов до цели.
*   **Завершенность:** Код (`tools/strategy/index.ts`) выглядит как **законченный математический движок**. Однако он функционирует как «советник» или «инструмент моделирования», который агент использует для оценки своих планов, а не как жестко зашитая логика поведения.

### D. Специализированные инструменты

1.  **Realtime:** Мост к мультимодальному интерфейсу. Генерирует подписанные URL для внешнего сервера (Web/Voice сессии), позволяя пользователю взаимодействовать с агентом голосом или через интерактивные HTML-страницы. **Зрелый функционал.**
2.  **Homework:** Глубокая интеграция с `NotebookLM` и специализированными промптами для образовательных задач. Умеет даже генерировать объясняющие видео. **Узкоспециализированный, но очень проработанный инструмент.**
3.  **Files / View:** Стандартный слой абстракции для работы с ФС в песочнице.
4.  **Utils:** Коллекция хелперов (дата, форматирование, работа со строками).
5.  **srcAgent:** Системный инструмент для вызова Gemini CLI. Особенность — **Hot-reload** кеша модулей Node.js, позволяющий системе применять изменения в коде без перезапуска.

### E. Устаревшие идеи и реформулировки

*   **«Cron-задачи»:** Понятие устарело. Система перешла к **Event/Sensor** модели. Heartbeat — это реакция на состояние мира/времени, а не просто запуск по таймеру.
*   **«Просто чат-бот»:** Архитектура `srcAgent` + `hot-reload` подтверждает, что система спроектирована как **самомодифицирующийся агент**.
*   **Память:** Идея «единого лога» заменена на **гибридную схему**: Граф (факты) + Векторная база (скиллы) + Key-Value (ноутпад). Это требует более точной формулировки в документации: «ACN1.0 использует триангуляцию памяти для обеспечения контекстной точности».

**Итог:** Система выглядит как зрелый фреймворк для автономных агентов с мощным аналитическим блоком (Strategy) и продвинутой системой проактивности (Heartbeat). Главная «фишка» кода — глубокая интеграция семантического поиска навыков непосредственно в цикл жизни сессии.
