# Memory Tool

Долговременная векторная память с семантическим поиском для хранения информации между сессиями.

## Описание

Инструмент `memory` использует векторные эмбеддинги Gemini для хранения и семантического поиска информации. Данные сохраняются между сессиями и могут использоваться для создания долговременной памяти агента.

## Возможности

- **Семантический поиск**: Поиск по смыслу, а не только по ключевым словам
- **Долговременное хранение**: Данные сохраняются в S3 или локально
- **Мультипользовательский режим**: Поддержка нескольких клиентов через `clientId`
- **Отслеживание сессий**: Автоматическое исключение уже просмотренных записей в рамках одной сессии
- **Векторные эмбеддинги**: Использует Gemini Embedding API
- **Косинусное сходство**: Точный поиск по векторному пространству

## API

### `memory.add(text, clientId?)`

Добавляет текст в память.

**Параметры:**
- `text` (string, обязательный) - Текст для сохранения
- `clientId` (string, опциональный) - Идентификатор клиента (по умолчанию: "default")

**Возвращает:** Сообщение об успехе с ID записи и общим количеством записей

**Пример:**
```javascript
await memory.add("Пользователь любит пиццу с ананасами");
// "Текст успешно добавлен в память. ID: 1234567890. Всего записей в памяти: 1"

await memory.add("Игнат работает программистом", "ignat");
// "Текст успешно добавлен в память. ID: 1234567891. Всего записей в памяти: 1"
```

### `memory.search(query, clientId?, sessionId?)`

Ищет в памяти по семантическому сходству.

**Параметры:**
- `query` (string, обязательный) - Поисковый запрос
- `clientId` (string, опциональный) - Идентификатор клиента (по умолчанию: "default")
- `sessionId` (string, опциональный) - Идентификатор сессии для отслеживания просмотренных записей

**Возвращает:** Отформатированный список найденных записей с процентами схожести

**Особенности с sessionId:**
- При указании `sessionId` найденные записи автоматически помечаются как просмотренные
- Повторные поиски в той же сессии не будут возвращать уже просмотренные записи
- Это предотвращает дублирование информации и засорение контекста

**Примеры:**
```javascript
// Обычный поиск без отслеживания
await memory.search("Что любит пользователь?");

// Поиск с отслеживанием сессии
await memory.search("Что любит пользователь?", "default", "session_123");
// Первый запрос вернёт результаты

await memory.search("Какие предпочтения пользователя?", "default", "session_123");
// Второй запрос НЕ вернёт уже просмотренные записи из первого запроса
```

### `memory.clearSession(sessionId)`

Очищает историю просмотренных записей для сессии.

**Параметры:**
- `sessionId` (string, обязательный) - Идентификатор сессии

**Возвращает:** Сообщение об успехе

**Пример:**
```javascript
await memory.clearSession("session_123");
// "История просмотров для сессии session_123 успешно очищена"
```

### `memory.getSessionStats(sessionId)`

Получает статистику по сессии (количество просмотренных записей).

**Параметры:**
- `sessionId` (string, обязательный) - Идентификатор сессии

**Возвращает:** Статистика сессии

**Пример:**
```javascript
await memory.getSessionStats("session_123");
// "Сессия session_123: просмотрено 5 записей"
```

## Особенности

### Порог схожести
Поиск возвращает только результаты с косинусным сходством >= 65%.

### Глобальный поиск
По умолчанию поиск выполняется по всем пользователям (всем `clientId`), если не указан конкретный клиент.

### Хранение данных
- **S3**: Основное хранилище (если настроено)
- **Локальное**: Fallback в папку `data/memory.{clientId}.json`

### Векторный индекс
Использует in-memory индекс с косинусным поиском для быстрого доступа к данным.

## Переменные окружения

```bash
# Gemini API
GEMINI_API_KEY=your_api_key_here
EMBEDDING_MODEL=gemini-embedding-001  # Опционально, по умолчанию

# S3 (опционально)
S3_BUCKET_NAME=your-bucket-name
S3_ENDPOINT=https://s3.amazonaws.com
S3_ACCESS_KEY_ID=your_access_key
S3_SECRET_ACCESS_KEY=your_secret_key
```

## Зависимости

- `@google/genai` - Gemini API
- `compute-cosine-similarity` - Расчет косинусного сходства
- `@aws-sdk/client-s3` - S3 клиент (опционально)
- `chalk` - Цветной вывод в консоль

## Структура данных

Каждая запись в памяти имеет следующую структуру:

```javascript
{
  id: string,              // Уникальный ID (timestamp)
  text: string,            // Текст записи
  embedding: number[],     // Вектор эмбеддинга
  timestamp: string,       // ISO timestamp
  meta: any                // Дополнительные метаданные (опционально)
}
```

## Примеры использования

### Сохранение информации о пользователе
```javascript
await memory.add("Анна работает в IT-компании, живет в Москве");
await memory.add("Анна любит читать научную фантастику");
await memory.add("Анна предпочитает работать удаленно");
```

### Поиск информации
```javascript
await memory.search("Где живет Анна?");
await memory.search("Что любит делать Анна в свободное время?");
await memory.search("Какой у Анны стиль работы?");
```

### Мультипользовательский режим
```javascript
// Пользователь 1
await memory.add("Мне нравится программировать на Python", "user1");

// Пользователь 2
await memory.add("Я предпочитаю JavaScript", "user2");

// Поиск только у пользователя 1
await memory.search("Что нравится пользователю?", "user1");
```

### Использование сессий для избежания дублирования
```javascript
const sessionId = "agent_session_001";

// Добавляем несколько записей
await memory.add("Пользователь любит путешествовать в горы");
await memory.add("Пользователь предпочитает активный отдых");
await memory.add("Пользователь работает удаленно");
await memory.add("Пользователь занимается йогой по утрам");

// Первый поиск в сессии
const result1 = await memory.search("Что любит делать пользователь?", "default", sessionId);
// Вернёт: "Пользователь любит путешествовать в горы", "Пользователь предпочитает активный отдых"

// Второй поиск в той же сессии
const result2 = await memory.search("Какие хобби у пользователя?", "default", sessionId);
// Вернёт ДРУГИЕ записи: "Пользователь занимается йогой по утрам"
// (исключит уже показанные в первом запросе)

// Проверка статистики
await memory.getSessionStats(sessionId);
// "Сессия agent_session_001: просмотрено 3 записей"

// Очистка сессии для начала заново
await memory.clearSession(sessionId);
```

## Производительность

- **Добавление**: ~1-2 секунды (зависит от API Gemini)
- **Поиск**: ~1-2 секунды для небольшой базы (< 1000 записей)
- **Масштабирование**: In-memory индекс подходит для баз до ~10,000 записей

## Ограничения

- Максимум 20 результатов в поиске
- Минимальный порог схожести 65%
- Поиск по всем пользователям может быть медленным для больших баз
