# Google Calendar Tool

Инструмент для управления Google Calendar через API.

## Возможности

- 📅 Просмотр событий на неделю и на конкретный день
- ➕ Добавление новых событий (разовых и повторяющихся)
- ✏️ Изменение существующих событий
- 🗑️ Удаление событий

## Настройка

### 1. Создание проекта в Google Cloud Console

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте новый проект или выберите существующий
3. В меню навигации выберите "APIs & Services" > "Library"
4. Найдите "Google Calendar API" и включите его

### 2. Создание OAuth 2.0 credentials

1. Перейдите в "APIs & Services" > "Credentials"
2. Нажмите "Create Credentials" > "OAuth client ID"
3. Выберите тип приложения: "Desktop app"
4. Укажите имя (например, "Calendar Tool")
5. Нажмите "Create"
6. Скачайте JSON файл с credentials

### 3. Установка credentials

1. Переименуйте скачанный файл в `credentials.json`
2. Поместите файл в директорию `tools/calendar/`
3. Путь должен быть: `tools/calendar/credentials.json`

### 4. Первый запуск и авторизация

При первом использовании инструмента:

1. Автоматически откроется браузер с формой авторизации Google
2. Выберите аккаунт Google
3. Разрешите доступ к календарю
4. Токен авторизации будет автоматически сохранён в `tools/calendar/calendar_token.json`

После этого инструмент будет работать автоматически без повторной авторизации.

## Использование

### Просмотр событий на неделю

```javascript
await calendar.viewWeek()
```

### Просмотр событий на конкретный день

```javascript
await calendar.viewDay('2025-10-15')
```

### Добавление события

```javascript
// Одно событие
await calendar.add(['monday'], 'Встреча с командой', '14:30', '01:00', false)

// Повторяющееся событие на несколько дней
await calendar.add(
  ['monday', 'wednesday', 'friday'],
  'Тренировка',
  '18:00',
  '01:30',
  true  // повторяется каждую неделю
)
```

### Изменение события

```javascript
// Изменить время события
await calendar.update('Встреча с командой', null, null, '15:00', '01:30', true)

// Изменить название
await calendar.update('Встреча с командой', null, 'Важная встреча', null, null, true)

// Изменить все события с этим названием (не только на этой неделе)
await calendar.update('Тренировка', null, 'Спортзал', '19:00', '01:00', false)
```

### Удаление события

```javascript
// ⚠️ ВАЖНО: Если не указать daysOfWeek (null), удалятся ВСЕ события с таким именем!

// Удалить события только в определенные дни на этой неделе
await calendar.delete('Встреча с командой', ['monday'], true)

// Удалить события в определенные дни (все будущие)
await calendar.delete('Тренировка', ['monday', 'friday'], false)

// ОПАСНО: Удалит ВСЕ события с этим названием на этой неделе
await calendar.delete('Встреча с командой', null, true)

// ОЧЕНЬ ОПАСНО: Удалит ВСЕ события с этим названием (все будущие)
await calendar.delete('Тренировка', null, false)
```

**⚠️ ПРЕДУПРЕЖДЕНИЕ:** Параметр `daysOfWeek` работает как фильтр безопасности. Если его не указать (`null`), будут удалены ВСЕ события с указанным именем в выбранном диапазоне. Рекомендуется всегда указывать конкретные дни недели для безопасного удаления.

## Формат параметров

### Дни недели (daysOfWeek)

Можно использовать английские или русские названия:
- `'monday'` / `'понедельник'`
- `'tuesday'` / `'вторник'`
- `'wednesday'` / `'среда'`
- `'thursday'` / `'четверг'`
- `'friday'` / `'пятница'`
- `'saturday'` / `'суббота'`
- `'sunday'` / `'воскресенье'`

### Время (startTime)

Формат: `"HH:MM"` (24-часовой формат)
- Примеры: `"09:00"`, `"14:30"`, `"18:45"`

### Длительность (duration)

Формат: `"HH:MM"`
- Примеры: `"01:00"` (1 час), `"01:30"` (1.5 часа), `"02:00"` (2 часа)

### Дата (date)

Формат: `"YYYY-MM-DD"`
- Примеры: `"2025-10-15"`, `"2025-12-31"`

## Часовой пояс

По умолчанию используется `Europe/Moscow`. Для изменения отредактируйте `timeZone` в коде.

## Напоминания

Все создаваемые события автоматически получают напоминание за 5 минут до начала (popup).

## Устранение неполадок

### Ошибка "credentials.json не найден"

Убедитесь, что файл `credentials.json` находится в `tools/calendar/credentials.json`

### Ошибка авторизации

1. Удалите файл `tools/calendar/calendar_token.json`
2. Запустите инструмент заново - откроется новое окно авторизации

### Ошибка "Таймаут авторизации"

Процесс авторизации занял более 5 минут. Попробуйте снова и завершите авторизацию быстрее.

## Файлы

- `index.js` - основной код инструмента
- `credentials.json` - OAuth 2.0 credentials (создаётся вручную)
- `calendar_token.json` - токен авторизации (создаётся автоматически)
