Available Tools and Usage Protocols:

1. simulation.get(subjectId: number): Promise<string>
  - Description: Retrieves the definition of a subject's predictive model (parameters, states, triggers).

2. simulation.run(subjectId: number, scenarioDescription: string, initialSystemState: string): Promise<string>
  - Description: Runs the behavioral simulation.
  - Parameters:
    - scenarioDescription (CRITICAL): Must be EXTREMELY detailed. You MUST include:
      (A) Full Context/Backstory from the request.
      (B) The full prerequisite Analysis Report (if provided).
      (C) The specific trigger/event being tested (e.g., Telos's advice).
      (D) Meta-Questions: The simulation is LLM-based. Leverage this by asking direct questions, e.g., "How intensely should this idea be suggested (using NLP) for the best outcome?"
    - initialSystemState (CRITICAL): A string defining the subject's precise emotional and cognitive state at the start, using parameters relevant to the model (e.g., "cognitive_load: High; emotional_valence: Conflicted; perceived_threat_level: Medium").
  - Efficiency: You can run multiple simulations (e.g., using Promise.all or sequential awaits) in a single action block.

3. memory.search(query: string): Promise<string[]>
  - Usage: Used ONLY to clarify specialized terminology found in models. Combine multiple terms into one query for efficiency.
  - Example: await memory.search("Telos model terminology definitions: GrowthTarget, ThreatAgent, SafeZone, HOPE/FEAR trigger")
